# 💻 Client-side Lifecycle (Vue Hooks)

_เกิดขึ้นเฉพาะบน Browser เท่านั้น (Safe for `window`, `document`, `localStorage`)_

### 🟢 `onMounted()`

> **หัวใจสำคัญของการทำ UI**

- **สถานะ:** Component ปรากฏบนหน้าจอ (DOM) เรียบร้อยแล้ว
- **การใช้งาน:** - เรียกใช้คำสั่ง Browser: `window`, `document`, `navigator`
  - ตั้งค่า Library ภายนอก: **Chart.js**, **Google Maps**, **Lottie**
  - เริ่มต้นทำ Animation: **GSAP**, **Anime.js**

### 🔴 `onUnmounted()`

- **สถานะ:** เมื่อ Component กำลังจะถูกทำลาย (เช่น ย้ายหน้า)
- **การใช้งาน:** - เคลียร์ Timer: `clearInterval`, `clearTimeout`
  - ยกเลิกการดักฟัง Event: `window.removeEventListener`
  - **ป้องกัน Memory Leak** ในแอปพลิเคชัน

## ⚠️ Common Pitfalls: The "Window" Error

ข้อผิดพลาดที่พบบ่อยที่สุดสำหรับมือใหม่คือการเรียกใช้ Browser API ผิดที่

| ❌ แบบที่ผิด (ทำให้ Error)        | ✅ แบบที่ถูก (ปลอดภัย)                                 |
| :-------------------------------- | :----------------------------------------------------- |
| วางไว้บนสุดของ `<script setup>`   | วางไว้ข้างใน `onMounted()`                             |
| `const width = window.innerWidth` | `onMounted(() => { width.value = window.innerWidth })` |

## 🧪 Workshop: ทดลองด้วยตัวเอง

ลองวางโค้ดนี้ในหน้า `pages/test.vue` เพื่อสังเกตลำดับการ Log ใน **Console** ของ Browser และ **Terminal** ของคุณ:

```typescript
<script setup>
    console.log('1. Setup: รันทั้ง Server และ Client')

    onMounted(() => {
        console.log('2. Mounted: รันเฉพาะ Client (Browser)')
    })

    onUnmounted(() => {
        console.log('3. Unmounted: รันเมื่อย้ายหน้า')
    })
</script>
```
