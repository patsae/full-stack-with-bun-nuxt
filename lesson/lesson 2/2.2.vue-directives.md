# 📘 Vue.js & Nuxt.js Core Concepts Reference

เอกสารสรุปพื้นฐานการใช้งาน Vue (Composition API) และ Nuxt สำหรับผู้เริ่มต้นจนถึงระดับกลาง

---

## Template Structure

โครงสร้างพื้นฐานของไฟล์ `.vue`

```vue
<template>
  <!--  ส่วน แสดงผล -->
</template>

<script setup>
// ส่วน Logic (Composition API)
</script>

<style scoped>
/* ส่วน CSS (เฉพาะไฟล์นี้) */
</style>
```

---

## 1. v-bind (Attribute Binding)

ใช้สำหรับเชื่อมต่อข้อมูลจาก Script เข้ากับ HTML Attributes

- **ตัวย่อ:** `:`
- **การใช้งาน:** ใช้กับ `src`, `href`, `disabled`, `class`, `style` และอื่นๆ

#### ตัวอย่างโค้ด templete:

```vue
<img v-bind:src="imagePath" />
<button :disabled="isPending">กดปุ่ม</button>
```

#### ตัวอย่างโค้ด script:

```typescript
const imagePath = "https://fastly.picsum.photos/id/1/5000/3333.jpg?hmac=Asv2DU3rA_5D1xSe22xZK47WEAN0wjWeFOhzd13ujW4";
const isPending = ref(false);
```

---

## 2. v-model (Two-way Data Binding)

เชื่อมต่อข้อมูลระหว่าง Input และตัวแปรแบบไป-กลับ (เปลี่ยนที่หนึ่ง อีกที่เปลี่ยนตาม)

#### ตัวอย่างโค้ด templete:

```vue
<input v-model="searchText" placeholder="พิมพ์..." />
<p>คุณกำลังค้นหา: {{ searchText }}</p>
```

#### ตัวอย่างโค้ด script:

```typescript
const searchText = ref("");
```

---

## 3. v-if & v-show (Conditional Rendering)

ใช้สำหรับแสดงผลตามเงื่อนไขที่กำหนด

- **v-if**: ลบ/เพิ่ม Element ออกจาก DOM จริงๆ (ประหยัด Resource ถ้าไม่ต้องสลับบ่อย)
- **v-show**: ใช้ CSS display: none (เหมาะสำหรับสิ่งที่ต้องสลับเปิด-ปิดบ่อยๆ)

#### ตัวอย่างโค้ด templete:

```vue
<p v-if="status === 'success'">บันทึกสำเร็จ</p>
<p v-else-if="status === 'error'">เกิดข้อผิดพลาด</p>
<p v-else>กำลังรอ...</p>

<div v-show="isVisible">ฉันถูกซ่อนด้วย CSS</div>
```

---

#### ตัวอย่างโค้ด script:

```typescript
const status = ref("success");
const isVisible = ref(false);
```

---

## 4. v-for (List Rendering)

ใช้สำหรับวนลูปแสดงผลข้อมูลจาก Array หรือ Object

- **สำคัญ**: ต้องมี `:key` เสมอเพื่อให้ Vue ติดตามสถานะได้ถูกต้อง

#### ตัวอย่างโค้ด templete:

```vue
<ul>
    <li v-for="item in items" :key="item.id">
      {{ item.name }}
    </li>
  </ul>
```

#### ตัวอย่างโค้ด script:

```typescript
const items = ref([
  { id: 1, name: "siripong" },
  { id: 2, name: "kwanchanok" },
  { id: 1, name: "mark" },
]);
```

## 5. Event & v-on (Event Handling)

ใช้สำหรับดักจับเหตุการณ์ต่างๆ เช่น การคลิก การพิมพ์

- **ตัวย่อ**: `@`

#### ตัวอย่างโค้ด templete:

```vue
  <div> {{ count }} </div>
  <button @click="count++">เพิ่มตัวเลข</button>
  <input @keypress.enter="alert('Hello Event')" ></input>
```

#### ตัวอย่างโค้ด script:

```typescript
const count = ref(0);
```

---

## 6. Methods (Functions)

ใน `<script setup>` เราจะใช้ฟังก์ชันปกติเพื่อจัดการ Logic ของโปรแกรม

#### ตัวอย่างโค้ด templete:

```vue
<div> {{ count }} </div>
<button @click="handleSubmit()">Submit</button>
```

#### ตัวอย่างโค้ด script:

```typescript
const handleSubmit = () => {
  console.log("submitted!");
};
```

---

## 7. CSS Binding

การจัดการ `Class` หรือ `Style` ด้วยเงื่อนไขทางโปรแกรม

#### ตัวอย่างโค้ด templete:

```vue
<div :class="{ active: isActive, error: hasError }">
    สถานะการใช้งาน
  </div>
```

#### ตัวอย่างโค้ด script:

```typescript
const isActive = ref(false);
const hasError = ref(true);
```

---

## 8. Computed Properties

ตัวแปรที่จะคำนวณค่าใหม่ `เฉพาะเมื่อ` ข้อมูลที่เกี่ยวข้องเปลี่ยนไป

#### ตัวอย่างโค้ด templete:

```vue
<button @click="price++">price ++</button>
<button @click="quantity++">quantity ++</button>
```

#### ตัวอย่างโค้ด script:

```typescript
const price = ref(100);
const quantity = ref(2);
const total = computed(() => price.value * quantity.value);
```

---

## 9. Watchers

ใช้เฝ้าติดตามการเปลี่ยนแปลงของตัวแปร และสั่งให้รัน Logic บางอย่าง (Side Effects)

#### ตัวอย่างโค้ด templete:

```vue
<button @click="count++">คลิก</button>
```

#### ตัวอย่างโค้ด script:

```typescript
const count = ref(0);
watch(count, (newId) => {
  console.log(newId);
});
```
