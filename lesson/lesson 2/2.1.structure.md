# โครงสร้าง Directory ของ Nuxt 3 (Nuxt Directory Structure)

Nuxt 3 มีการจัดเตรียมโครงสร้างโฟลเดอร์ที่เป็นมาตรฐาน ซึ่งช่วยให้การพัฒนาเว็บแอปพลิเคชันเป็นไปอย่างมีระบบ และง่ายต่อการดูแลรักษา (Scalable) นี่คือส่วนประกอบสำคัญต่างๆ ที่คุณจะพบเมื่อสร้างโปรเจกต์ใหม่:

```text
nuxt-app/
├── .nuxt/               # โฟลเดอร์ที่ Nuxt สร้างขึ้นอัตโนมัติเมื่อรันโปรเจกต์ (ไม่ต้องแก้ไข)
├── assets/              # ไฟล์ทรัพยากร (CSS, Sass, รูปภาพที่ต้องผ่านเครื่องมือ Build เช่น Vite)
├── components/          # Vue components ที่ใช้ซ้ำได้ (Auto-imported อัตโนมัติ)
├── composables/         # Composition API functions สำหรับจัดการ Logic (Auto-imported)
├── content/             # สำหรับสร้างเนื้อหาด้วย Markdown, JSON, YAML (ใช้ร่วมกับโมดูล Nuxt Content)
├── layouts/             # โครงร่างเทมเพลตของหน้าเว็บ (เช่น Header, Footer, Sidebar)
├── middleware/          # โค้ด Route Middleware ที่ทำงานก่อนจะเกิดการเปลี่ยนหน้า
├── node_modules/        # แพ็กเกจและ Dependencies ต่างๆ (สร้างโดย npm/yarn/bun)
├── pages/               # หน้าเว็บแต่ละหน้า (ทำ File-based routing อัตโนมัติ)
├── plugins/             # ปลั๊กอิน Vue แจ้งให้ทำงานตอนเริ่มต้นแอปพลิเคชัน
├── public/              # ไฟล์สถิตย์ (Static files เช่น robots.txt, favicon) ที่ไม่ผ่านการประมวลผล
├── server/              # ส่วนรับฝั่ง Server-side (API Routes, Server Middleware)
│   ├── api/             # สำหรับสร้าง API endpoints (เข้าถึงผ่าน /api/*)
│   ├── routes/          # สำหรับสร้าง Server routes (เข้าถึงผ่าน /*)
│   └── middleware/      # Server middleware ที่ทำงานในทุกๆ request
├── utils/               # ฟังก์ชันช่วยเหลือ (Helper/Utility functions) (Auto-imported)
├── .gitignore           # กำหนดไฟล์ที่ไม่ต้องการจัดเก็บใน Git
├── app.vue              # Component หลัก (Root Component) ของแอปพลิเคชัน
├── nuxt.config.ts       # ไฟล์ตั้งค่าหลักและคอนฟิกต่างๆ ของ Nuxt
├── package.json         # กำหนดชื่อโปรเจกต์ สคริปต์ และ dependencies
└── tsconfig.json        # ตั้งค่าให้กับ TypeScript
```

## คำอธิบายโฟลเดอร์และไฟล์ที่สำคัญ

### 1. `app.vue`
เป็น Component หลักที่เป็นเส้นเลือดใหญ่ของแอปพลิเคชัน ซึ่งครอบคลุมหน้าจอทั้งหมด ส่วนใหญ่จะใช้สำหรับการเปิดเพจด้วยคอมโพเนนต์ `<NuxtLayout>` และ `<NuxtPage>`

### 2. `pages/` (ระบบจัดหน้าอัตโนมัติ)
หัวใจสำคัญของการทำ Routing ในระบบ Nuxt เพียงแค่เราสร้างไฟล์ `.vue` ไว้ในโฟลเดอร์นี้ ฝั่ง Nuxt จะจัดการสร้าง Route ผูกเข้ากับ URL ให้โดยอัตโนมัติตามชื่อไฟล์ (File-based Routing) ควบคู่กับ Vue Router
*ตัวอย่าง: การสร้างสร้างไฟล์ `pages/about.vue` จะเผยแพร่ไปยังหน้า `https://domain.com/about` โดยอัตโนมัติ*

### 3. `components/`
พื้นที่เก็บ UI Component สำหรับ Vue ทั้งหมด หากเราสร้างคอมโพเนนต์ในพื้นที่นี้ Nuxt จะทำ **Auto-import** ให้อัตโนมัติ หมายความว่าเราสามารถเรียกใช้งาน component ในไฟล์ `.vue` หน้าอื่นได้ทันทีโดยไม่ต้องเขียนโค้ด `import MyComponent from ...` ให้เสียเวลา

### 4. `layouts/`
ใช้ในการสร้างเทมเพลตโครงสร้างหลักของหน้าเว็บ (Layouts) ยกตัวอย่างเช่น บางหน้าจอมี Navbar และ Footer รูปแบบเดียวกันหมด ช่วยอำนวยความสะดวกในการจัดระเบียบและลดความซ้ำซ้อนของการเขียนโค้ด

### 5. `composables/` และ `utils/`
*   **composables/**: สำหรับเก็บส่วนประกอบ Custom Composition API (ส่วนใหญ่ตั้งชื่อด้วยการใช้ `use...` นำหน้า) ระบบของ Nuxt จะทำ Auto-import ให้เช่นกัน
*   **utils/**: ทำงานคล้ายคลึงกับ composables แต่จะถูกออกแบบมาไว้เพื่อมุ่งเน้นเก็บ ฟังก์ชันช่วยเหลือทั่วไป (Utility / Helper functions)

### 6. `public/` และ `assets/`
*   **public/**: จะสงวนไว้เก็บไฟล์ที่ไม่ต้องการผ่านการประมวลผลโดยเครื่องมือ Build ไฟล์เหล่านี้จะถูกเข้าถึงไดเรกทอรีเว็บไซต์ได้โดยตรง เช่น `/favicon.ico` หรือรูปภาพดิบๆ ที่ไม่ต้องดัดแปลง
*   **assets/**: มุ่งเน้นการเก็บไฟล์ที่ต้องมีการประมวลผลก่อนการนำไปใช้งานบนเบราว์เซอร์ เช่น การคอมไพล์ SCSS ไปเป็น CSS หรือการย่อขนาดไฟล์รูปภาพ รูปภาพโลโก้ เป็นต้น

### 7. `server/`
โฟลเดอร์สุดทรงพลังในโครงสร้างของ Nuxt 3 (ซึ่งพัฒนาด้วยเอนจินของทาง Nitro) มันช่วยให้แอปพลิเคชันสามารถสร้าง Backend API ของตนเองได้ด้วย
*   โดยการสร้างไฟล์ซ้อนไว้ใน `server/api/` คุณสามารถจะใช้แอปฯ ของเราเป็น RESTful API เข้าถึงได้ผ่าน `/api/...` เลย!

### 8. `nuxt.config.ts`
คอนฟิกไฟล์และจุดศูนย์กลางในการปรับแต่งค่าติดตั้ง Nuxt ทั้งระบบ เช่น การกำหนดยัด Title, Meta tags, การเพิ่มโมดูลเสริมต่างๆ (อย่าง TailwindCSS, Nuxt Image), การโหลดฟอนต์ต่างๆ ไปจนถึงการตั้งค่า Environment variables

---
**💡 Tips / คำแนะนำ:** โฟลเดอร์ `.nuxt` เป็นโฟลเดอร์ที่ระบบสร้างขึ้นอัตโนมัติ โปรด **หลีกเลี่ยงการเข้าไปแก้ไขโค้ดในโฟลเดอร์นี้โดยตรง** เพราะเมื่อใดที่มีการรันคำสั่ง run หรือ build ขึ้นมาใหม่ ข้อมูลในโฟลเดอร์จะถูกสร้างขึ้นเพื่อเขียนทับใหม่ทันที
