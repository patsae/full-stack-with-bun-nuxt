# Route Middleware ใน Nuxt 3

ใน Nuxt 3 เราสามารถใช้ **Route Middleware** เพื่อดักจับหรือจัดการการทำงานทุกครั้งที่เปลี่ยนหน้า (Navigation) เช่น การตรวจสอบสิทธิ์ผู้ใช้ (Authentication), การเปลี่ยนเส้นทางอัตโนมัติ (Redirect), หรือการเก็บสถิติ เป็นต้น

---

## การสร้าง Global Middleware (ทำงานทุกหน้าโดยอัตโนมัติ)

จากการสร้างไฟล์ `app/middleware/default.global.ts` เราได้ใส่โค้ดดังนี้:

```typescript
export default defineNuxtRouteMiddleware((to, from) => {
  console.log(`[Middleware] Navigating from: ${from.path} to: ${to.path}`)
})
```

### คำอธิบายการทำงาน

1. **`defineNuxtRouteMiddleware`**
   เป็นฟังก์ชัน (Auto-imported) ที่ใช้สำหรับประกาสขอบเขต (define) ตัว Middleware ใน Nuxt 3 ซึ่งจะรับ Parameter หลักคือ Callback function ที่มี `to` และ `from`

2. **`to` และ `from`**
   - **`from`:** ออบเจ็กต์ที่เก็บข้อมูลของเส้นทาง (Route) ก่อนที่จะเปลี่ยนมา (หน้าปัจจุบันที่กำลังจะจากไป)
   - **`to`:** ออบเจ็กต์ที่เก็บข้อมูลของเส้นทางเป้าหมาย (หน้าที่กำลังจะไปแสดงผล)
   ทั้งคู่จะสามารถเข้าถึง property ต่างๆ ของเส้นทางได้ เช่น `.path`, `.fullPath`, `.name`, `.query`, หรือ `.params`

3. **`console.log()`**
   เมื่อเราทำการสลับหน้าเว็บไปมา ระบบจะทำการสั่งรันคำสั่งภายใต้ Middleware นี้ก่อนที่จะโหลดหน้าเป้าหมายเสร็จ ทำให้เราจะเห็น Log พิมพ์ออกมาใน Terminal (สำหรับฝั่ง Server) และ Console ใน Browser (หากรันฝั่ง Client) บอกว่ามา **จากที่ไหน (from.path)** กำลังเปลี่ยน **ไปที่ไหน (to.path)**

4. **การตั้งชื่อไฟล์ด้วยลักษณนาม `.global.ts`**
   การตั้งชื่อไฟล์จบด้วย `.global.ts` (เช่น `default.global.ts`) จะเป็นการบอกให้ Nuxt รับรู้ว่า Middleware ตัวนี้ **ให้บังคับใช้กับทุกหน้า (Global)** โดยอัตโนมัติ ซึ่งหากเราไม่ใส่ `.global` การเรียกใช้ Middleware ตัวนี้จะต้องไปประกาศในหน้าที่ต้องการดักจับเองด้วยการตั้งค่า `definePageMeta({ middleware: 'default' })`

---

**สรุปประโยชน์:** การเขียน Middleware จะมีประโยชน์มากเวลาที่เราต้องการควบคุม Flow การเข้าถึงหน้าเว็บต่างๆ ของแอปพลิเคชันอย่างเป็นระบบครับ
