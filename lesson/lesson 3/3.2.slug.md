# การทำ Dynamic Route ด้วย Slug (Param)

ใน Nuxt 3 การสร้าง "Dynamic Route" (หน้าเว็บที่มี URL ดิ้นได้ตามข้อมูล หรือรับพารามิเตอร์) จะใช้การตั้งชื่อไฟล์หรือโฟลเดอร์ด้วยวงเล็บก้ามปู (Bracket) อย่างเช่น `[slug].vue` เป็นแนวทางหลัก ซึ่งจากตัวอย่างที่คุณดู มีการส่งและรับค่าพารามิเตอร์ 2 รูปแบบดังนี้ครับ

---

## 1. แบบพารามิเตอร์ตัวเดียว (Single Slug)

**โครงสร้างไฟล์:** `app/pages/routing/[slug].vue`
รูปแบบนี้ใช้เมื่อต้องการหน้าเว็บที่รับค่าพารามิเตอร์เพียงค่าเดียว เช่น ID ของหน้า, Username, ชื่อบทความ ฯลฯ

**ตัวอย่าง URL:** `/routing/my-first-post` หรือ `/routing/123`

**การทำงานในไฟล์ `[slug].vue`:**
```vue
<script setup lang="ts">
  // useRoute เป็น Auto-imported composable สำหรับดูข้อมูล URL ปัจจุบัน
  const route = useRoute()
  
  // ดึงค่า slug (โดยชื่อ 'slug' จะต้องตรงกับชื่อไฟล์ [slug].vue)
  const slug = route.params.slug
</script>
```
**อธิบาย:** เมื่อผู้ใช้เข้าสู่เส้นทางดังกล่าว เช่น `/routing/hello-world` ส่วนของคำว่า `hello-world` จะถูกเก็บลงในออบเจ็กต์ `route.params` ด้วย key (ชื่อตัวแปร) ที่ตรงกับชื่อไฟล์ที่เราครอบด้วย `[]` ซึ่งในที่นี้คือ `slug`

---

## 2. แบบพารามิเตอร์หลายตัว (Multiple Slugs)

**โครงสร้างไฟล์:** `app/pages/routing/two-slug/[idx]/[name].vue`
ในกรณีที่เราจำเป็นต้องรับหลายตัวแปรใน 1 เส้นทาง ตัว Nuxt อนุญาตให้เราตั้งชื่อโฟลเดอร์เป็น Param ได้ด้วยครับ

**ตัวอย่าง URL:** `/routing/two-slug/99/john-doe`

**การทำงานในไฟล์ `[name].vue`:**
```vue
<script setup lang="ts">
  const route = useRoute()
  
  // ระบบจะอ่านค่าจาก โฟลเดอร์ [idx] และ ชื่อไฟล์ [name].vue มาให้พร้อมกัน
  const idx = route.params.idx
  const name = route.params.name
</script>
```
**อธิบาย:** 
- เมื่อผู้ใช้เข้า URL `/routing/two-slug/99/john-doe` 
- ค่า `99` จะถูกเก็บลงในตัวแปร `idx` (เพราะอยู่ถูกตำแหน่งกับชื่อโฟลเดอร์ `[idx]`)
- ค่า `john-doe` จะถูกเก็บลงในตัวแปร `name` (เพราะถูกตำแหน่งกับชื่อไฟล์ `[name].vue`)
- ซึ่งทั้งหมดนี้ถูกรวมมาให้แล้วใน `route.params`

**สรุป:** Nuxt 3 อ่านชื่อโฟลเดอร์และชื่อไฟล์ที่มี `[]` ครอบเป็นตัวแปร (params) ให้อัตโนมัติ ทำให้เราเขียนรับค่าต่อยอดเชื่อมต่อกับ Database หรือ API ได้โดยง่ายมากๆ เลียนแบบการเขียน Path เลยครับ
