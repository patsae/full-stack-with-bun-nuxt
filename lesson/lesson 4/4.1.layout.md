# การใช้งาน Layouts ใน Nuxt 3

Nuxt 3 มีระบบ "Layout" ซึ่งเปรียบเสมือน Template หรือกรอบหน้าต่างหลักที่ครอบหน้าเว็บแต่ละหน้า (Pages) เอาไว้อีกชั้น ช่วยให้เราสามารถสร้างส่วนประกอบที่ใช้ซ้ำกันหลายหน้า เช่น `Header`, `Sidebar` หรือ `Footer` ได้อย่างเป็นระเบียบ

จากการสร้างไฟล์ 2 แบบใน `app/layouts/` เราสามารถสรุปการทำงานได้ดังนี้

---

## 1. Default Layout (`app/layouts/default.vue`)

ไฟล์ Layout ที่ตั้งชื่อให้ว่า **`default.vue`** จะถูกมองว่าเป็น Layout มาตรฐานหลักของแอปพลิเคชัน นั่นหมายความว่า ถ้าหน้าใดๆ (ในโฟลเดอร์ `pages`) **ไม่ได้** มีการตั้งค่าระบุให้ไปใช้ Layout ตัวไหนเป็นพิเศษ หน้านั้นๆ ก็จะถูกครอบด้วยไฟล์ `default.vue` นี้แบบอัตโนมัติ 

**หัวใจสำคัญ:** ภายในไฟล์ Layout (ทั้ง Default และ Custom) จะต้องมีการเรียกใช้คอมโพเนนต์ `<slot />` เสมอ เพื่อให้เป็นตำแหน่งในการแทรกเนื้อหาของแต่ละหน้า (`<NuxtPage />`) ลงไป

---

## 2. Custom Layout (`app/layouts/custom.vue`)

ในบางเว็บเพจ เราอาจไม่ต้องการใช้ Header หรือ โครงสร้างหลักเหมือนหน้าอื่นๆ เช่น หน้า Admin Dashboard, หน้า Login ผู้ใช้ หรือหน้าแสดงข้อผิดพลาด (Error Page) เราสามารถสร้าง Layout เฉพาะกิจขึ้นมาด้วยการ **ตั้งชื่อใหม่** เช่น `custom.vue` หรือ `auth.vue` หรือ `admin.vue` ก็ได้

การเรียกใช้งาน Custom Layout สามารถทำได้โดยไปเรียกใช้ฟังก์ชัน `definePageMeta()` ลิขสิทธิ์เฉพาะของ Nuxt ภายในหน้า (.vue) ที่ต้องการ ดังตัวอย่างที่มีการตั้งค่าไว้ในไฟล์ `app/pages/layouts-demo.vue`:

```vue
<script setup lang="ts">
// เป็นการเจาะจงว่าหน้า layout-demo.vue นี้นะ ให้ใช้กรอบที่ชือ custom.vue !
definePageMeta({
  layout: 'custom' 
})
</script>
```

### สรุปข้อแตกต่างและการใช้งาน:
1. **`default.vue`**: เมื่อเราเพิ่มไฟล์นี้แล้ว Nuxt จะนำไปครอบทุกๆ `Page` อัตโนมัติ จึงเหมาะกับการทำแถบนำทาง (Navbar) หรือส่วนท้ายกระดาษ (Footer) หลักของเว็บ
2. **`[ชื่อเจาะจง].vue`**: เราเอาไปใช้สำหรับหน้าใหม่ที่จำเป็นต้องแสดง Template โครงสร้างแตกต่างไปจากหน้าปกติ โดยจะต้องไปเขียน `definePageMeta({ layout: '[ชื่อเจาะจง]' })` ที่หน้าเป้าหมายอยู่เสมอ
