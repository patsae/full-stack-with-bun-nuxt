

# Setup Project

### 🛠️ ติดตั้ง Bun.js

เปิด Powershell ด้วยสิทธิ Administrator จากนั้นพิมพ์คำสั่งติดตั้ง Bun.js

```
powershell -c "irm bun.sh/install.ps1 | iex"
```

ทดสอบหลังการติดตั้ง พิมพ์คำสั่ง bun -v หรือ bun --version เพื่อตรวจดูเวอร์ชั่น

```
bun -v
```

### 🛠️ ติดตั้ง Nuxt.js

```
bun create nuxt@latest nuxt-learning-hub

Which template would you like to use?
- minimal – Minimal setup for Nuxt 4

Which package manager would you like to use?
- bun

Initialize git repository?
- yes

Would you like to browse and install modules?
- No

cd nuxt-learning-hub
bun run dev
```

### 🛠️ Nuxt Configuration

หลังจากที่ติดตั้งเสร็จแล้ว ต่อไปมาตั้งค่า Nuxt Configuration ได้ที่ไฟล์ nuxt.config.ts โดยปกติไฟล์นี้จะอยู่ที่ Root ของโปรเจคเสมอ หน้าที่หลักคือการ Override (ทับ) หรือ Extend (ขยาย) ความสามารถพื้นฐานของ Nuxt ให้ตรงตามความต้องการของเรา

1. สร้างไฟล์ .env.dev

```
APP_NAME="Nuxt Learning Hub"
BASE_URL=/
```

2. ติดตั้ง package @types/node

```
bun i --save-dev @types/node
```

3. ปรับปรุง package.json

```
"scripts": {
    ...
    "dev": "bun --env-file=.env.dev nuxt dev",
  },
```

4. ตั้งค่า nuxt.config.ts

```
export default defineNuxtConfig({
  compatibilityDate: "2025-07-15",
  devtools: { enabled: false },
  ssr: false,
  nitro: {
    preset: "bun",
  },
  app: {
    baseURL: process.env.BASE_URL,
    head: {
      title: "Nuxt Learning Hub",
      meta: [
        {
          name: "description",
          content: "Full Stack with Bun & Nuxt.js",
        },
      ],
    },
  },
});
```
