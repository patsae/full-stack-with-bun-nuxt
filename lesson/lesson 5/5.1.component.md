# 🧱 Components คืออะไร?

**Component** คือ ส่วนประกอบย่อยๆ ของหน้าเว็บ (UI) ที่สามารถแยกออกมาเป็นชิ้นๆ ได้เหมือนกับตัวต่อเลโก้ (Lego) ซึ่งแต่ละชิ้นสามารถทำงานได้ด้วยตัวเอง และสามารถนำกลับมาใช้งานซ้ำได้ (Reusable) ในหลายๆ ส่วนของแอปพลิเคชัน 

ใน Nuxt 3 / Vue 3 นั้น Component มักจะถูกเขียนอยู่ในไฟล์นามสกุล `.vue` (Single-File Component) ซึ่งภายในหนึ่งไฟล์จะประกอบไปด้วย 3 ส่วนหลัก ได้แก่:

1. `<template>` สำหรับเขียนโค้ด HTML เพื่อกำหนดโครงสร้างและหน้าตาของคอมโพเนนต์
2. `<script setup>` สำหรับเขียน JavaScript / TypeScript เพื่อกำหนดการทำงานและเก็บข้อมูล
3. `<style>` สำหรับเขียน CSS เพื่อตกแต่งความสวยงาม (เลือกใส่หรือไม่ใส่ก็ได้)

## การทำงานและข้อดีของ Component

- **สร้างครั้งเดียว ใช้ได้หลายที่ (Reusability):** แทนที่เราจะเขียนโค้ด HTML เช่น โครงสร้างปุ่มที่ซับซ้อน ซ้ำๆ กันหลายจุด เราสามารถจัดโครงสร้างนั้นสร้างเป็น 1 Component แล้วนำไปเรียกใช้ (Import) ได้ทุกหน้าเลย
- **แยกส่วนให้จัดการง่าย (Maintainability):** โค้ด HTML หรือ Logic ที่ยาวมากๆ เมื่อแบ่งเป็นส่วนย่อยๆ จะทำให้อ่านง่ายและแก้ไขรักษาได้ง่ายขึ้น เมื่อมีปัญหาหรือต้องการปรับปรุงการแสดงผล ก็แค่เข้าไปแก้ไขเฉพาะ Component เรื่องนั้นๆ โดยไม่กระทบหน้าอื่น
- **ทำงานเป็นอิสระต่อกัน (Encapsulation):** ข้อมูลภายในสถานะ (State) หรือตัวแปรภายใน Component จะถูกแยกเป็นอิสระจาก Component อื่นๆ ซึ่งกันและกัน

**ยกตัวอย่างเช่น:** 
ถ้าเรามี "ปุ่ม" ที่ตกแต่งมาอย่างดีและถูกใช้งานใน 10 หน้าย่อย แทนที่จะเขียนโค้ด `<button class="btn btn-primary ...">...</button>` ไปทุกๆ หน้า ก็เปลี่ยนเป็นสร้างไฟล์ `MyButton.vue` แล้วนำไปเรียกใช้ด้วยแท็ก `<MyButton />` แทน หากในอนาคตต้องการเปลี่ยนสีปุ่ม ก็เข้ามาแก้ที่ `MyButton.vue` ที่เดียวเท่านั้น
