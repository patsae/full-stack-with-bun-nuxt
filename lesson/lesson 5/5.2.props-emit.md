# 🔌 Props & Emit (Communication)

เนื่องจาก Components แต่ละตัวถูกออกแบบให้แยกการทำงานออกจากกันได้อย่างอิสระ แต่ในความเป็นจริง Component ย่อยๆ มักจะมีความจำเป็นเพื่อนำข้อมูลจาก Component หลักหรือที่อื่นเข้ามาใช้งานด้วย รวมถึงเมื่อผู้ใช้กระทำการอะไรบางอย่างก็จะต้องมีการบอกให้ Component หลักรับรู้ ซึ่งเราสามารถจัดการการสื่อสารนี้ได้ด้วย **Props** และ **Emit**

* **Props**: เปรียบเสมือน "ท่อน้ำ" ที่ใช้สำหรับ **ส่งข้อมูล** จากคอมโพเนนต์ **แม่ (Parent)** ทะลุผ่านลงไปสู่คอมโพเนนต์ **ลูก (Child)**
* **Emit**: เปรียบเสมือน "ลำโพง" ที่ใช้สำหรับ **ส่งสัญญาณ/แจ้งเตือน** การกระทำจากคอมโพเนนต์ **ลูก (Child)** ย้อนกลับขึ้นไปบอกคอมโพเนนต์ **แม่ (Parent)**

---

## 1. Props (ส่งข้อมูลจากบนลงล่าง แม่ -> ลูก)

เป็นการสื่อสารแบบทางเดียว (One-way Data Flow) เมื่อคอมโพเนนต์แม่ต้องการจะส่งข้อมูล หรือตัวแปรบางอย่าง ให้กับคอมโพเนนต์ลูกเพื่อนำไปแสดงผลหรือประมวลผล

### `app/pages/props-emit/parentComponent.vue`
ในหน้าของแม่ (Parent) เราเรียกใช้งาน `<ChildComponent />` และสามารถส่งมอบค่าโดยเขียนเป็น attribute แนบไป (ตัวอย่างนี้เราตั้งชื่อข้อมูลที่ส่งไปว่า `message`)
```vue
<template>
    <!-- แม่เรียกใช้ลูก และส่งข้อความไปหาลูกผ่าน Props ชื่อ message -->
    <ChildComponent message="สวัสดีลูกรัก" />
</template>

<script setup lang="ts">
    // นำเข้า (Import) คอมโพเนนต์ลูกเพื่อมาแสดงผล
    import ChildComponent from './childComponent.vue';
</script>
```

### `app/pages/props-emit/childComponent.vue`
ในหน้าของลูก (Child) แม้จะมีข้อมูลส่งมา แต่การจะนำมาใช้ได้เราจะต้อง "ประกาศรับข้อมูล" เสียก่อน ผ่านคีย์เวิร์ดที่เรียกว่า `defineProps` 
```vue
<template>
    <!-- ลูกเอาข้อมูล prop "message" ที่ได้รับ ออกมานำเสนอหรือใช้งาน -->
    <p>{{ props.message }}</p>
</template>

<script setup lang="ts">
    // ลูกใช้ defineProps ประกาศรับข้อมูลชื่อ "message" เข้ามา พร้อมระบุว่าเป็นประเภท String (ข้อความ)
    const props = defineProps<{
         message: string
    }>()
</script>
```

## 2. Emit (ส่งสัญญาณจากล่างขึ้นบน ลูก -> แม่)

เมื่อคอมโพเนนต์ลูก (Child) เกิดการกระทำบางอย่างที่สำคัญ (เช่น ผู้ใช้กดปุ่ม หรือพิมพ์ข้อมูลลงในช่อง) แล้วจำเป็นที่จะต้องสะกิดบอกให้คอมโพเนนต์แม่ (Parent) รับทราบ หรือในบางครั้งอาจจะแนบข้อมูลบางส่วนสะท้อนกลับไปให้แม่ด้วย

### `app/pages/props-emit/childComponent.vue`
ในหน้าของลูก (Child) เราจะเชื่อมต่อกับการกระทำบน template และใช้ฟังก์ชัน `defineEmits` ในการประกาศสัญญาณที่เราสามารถตะโกนบอกได้
```vue
<template>
    <!-- เมื่อผู้ใช้คลิกที่ปุ่ม ให้เข้าไปทำงานที่ฟังก์ชัน notifyParent -->
    <button @click="notifyParent">ส่งค่าให้แม่</button>
</template>

<script setup lang="ts">
// 1. ประกาศท่อส่งสัญญาณ (Event) โดยตั้งชื่อว่า 'update'
const emit = defineEmits(['update'])

const notifyParent = () => {
  // 2. เมื่อฟังก์ชันทำงาน ให้ยิงสัญญาณ 'update' แจ้งแม่ พร้อมส่งข้อมูลเป็นเลข 100 กลับไปด้วยเพื่อเป็นของติดไม้ติดมือ
  emit('update', 100)
}
</script>
```

### `app/pages/props-emit/parentComponent.vue`
ในคอมโพเนนต์แม่ (Parent) เมื่อมีการเรียกใช้งานหน้าลูกไว้ เราสามารถเลือก "ดักฟังสัญญาณ" ของลูกได้เลย ผ่านการใส่โค้ด `@ชื่อเหตุการณ์` ในแท็กของลูก 
```vue
<template>
    <!-- แม่ดักฟังเหตุการณ์ที่ชื่อ `@update` จากลูก เมื่อได้รับสัญญาณให้เรียกใช้งานฟังก์ชัน handleUpdate -->
    <ChildComponent @update="handleUpdate" />
</template>

<script setup lang="ts">
    import ChildComponent from './childComponent.vue';
    
    // เมื่อคอมโพเนนต์ลูกยิงสัญญาณมา ฟังก์ชันนี้จะทำงาน โดยมีพารามิเตอร์ (value) สำหรับรับค่า 100 ที่ลูกส่งแนบมาให้
    const handleUpdate = (value) => {
        // แม่สามารถนำ value ที่ส่งคืนกลับมาไปใช้งานต่อได้
        console.log('แม่ได้รับค่าแล้ว:', value)
    }
</script>
```
