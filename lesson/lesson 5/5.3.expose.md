# 🔓 Expose & Ref (การสั่งงานลูกจากภายนอก)

อีกหนึ่งวิธีการที่คอมโพเนนต์แม่ (Parent) จะสื่อสารกับคอมโพเนนต์ลูก (Child) นอกเหนือจากการส่งข้อมูลผ่านทาง Props ไปเรื่อยๆ ก็คือ การเจาะเข้าไป "สั่งงาน" ฟังก์ชันของอ็อบเจ็กต์ลูกโดยตรง ผ่านการใช้ **Template Ref** ร่วมกับ **`defineExpose`** 

เทคนิคนี้มักใช้งานได้ดีในกรณีที่คอมโพเนนต์แม่ไม่ได้ต้องการส่งหรือเปลี่ยนค่าข้อมูล (Data) ไปทิ้งไว้ที่ลูก ทว่าต้องการให้ลูกลงมือทำงานบางอย่างที่ลูกเองมีวิธีการอยู่แล้วในจังหวะพิเศษ เช่น การสั่งรีเซ็ตฟอร์ม (Reset Form), เปิดหน้าต่างป๊อบอัพแบบ Modal หรือการเลื่อนตำแหน่งในหน้า (Scroll to View)

---

## 1. การเตรียมพร้อมที่ฝั่งลูก (Child) ด้วย `defineExpose`

โดยปกติแล้ว ในการใช้งาน `<script setup>` ของ Vue 3 ตัวแปรและฟังก์ชันทุกชิ้นภายในไฟล์คอมโพเนนต์จะถือว่าเป็น **"ส่วนตัว" (Private)** โดยสมบูรณ์ ซึ่งคอมโพเนนต์อื่นไม่สามารถมาเรียกใช้งานสุ่มสี่สุ่มห้าได้ จนกว่าเราจะ "ประกาศให้อนุญาต" อย่างเป็นทางการผ่านคำสั่ง `defineExpose`

### ตัวอย่างแทรกใน `childComponent.vue`
```vue
<script setup lang="ts">
// 1. สร้างฟังก์ชันการทำงานที่ต้องการให้ใครสักคนจากภายนอกสามารถเรียกได้
const showAlert = () => {
    alert('🚨 แจ้งเตือนจาก Child Component: ข้าถูกเรียกใช้งาน!');
};

// 2. นำข้อมูล ตัวแปร หรือฟังก์ชันนั้นๆ มาลงทะเบียนเปิดเผยสิทธิ์
defineExpose({
    showAlert
});
</script>
```

## 2. การสร้างจุดเชื่อมต่อและเรียกใช้งานจากฝั่งแม่ (Parent)

เมื่อลูกเปิดเผย (Expose) สิทธิ์ให้เรียกใช้งานแล้ว ฝั่งแม่จำเป็นจะต้องจับเป้าหมาย (Reference) ไปยังคอมโพเนนต์ลูกตัวนั้นให้ได้ก่อน เพื่อให้ระบบรู้ว่าแม่กำลังจะสั่งงานลูกคนไหน 

เราทำสิ่งนั้นได้โดยการกำหนด Attribute แบบพิเศษที่ชื่อว่า `ref="..."` ไว้ที่คอมโพเนนต์ของลูกในตอนปั้นแท็ก (Render) และจากนั้นสร้างตัวแปรในฝั่ง `<script>` ที่มีชื่อตรงกันเป๊ะๆ ขึ้นมาเป็นตัวรับ

### ตัวอย่างแทรกใน `parentComponent.vue`
```vue
<template>
    <button @click="triggerChildAlert">เรียกใช้งานฟังก์ชันของลูก (Expose)</button>

    <!-- 1. คล้องชื่อให้ตัวแปร childRef ชี้อ้างอิงเป้าหมายมายังคอมโพเนนต์ลูกตัวนี้ -->
    <ChildComponent ref="childRef" />
</template>

<script setup lang="ts">
import { ref } from 'vue';
import ChildComponent from './childComponent.vue';

// 2. สร้างตัวแปรมาคอยรับค่า Reference ของคอมโพเนนต์ลูก 
const childRef = ref<InstanceType<typeof ChildComponent> | null>(null);

// 3. สร้างฟังก์ชันฝั่งแม่ เพื่อผูกชัตเตอร์เข้ากับปุ่มกด
const triggerChildAlert = () => {
    // เราควรตรวจสอบก่อนเสมอว่าลูกปรากฏแน่ชัด (จับ Reference สำเร็จ)
    if (childRef.value) {
        // เมื่อพร้อมแล้ว ก็สั่งเรียกฟังก์ชัน showAlert() ที่ลูกลงทะเบียน Expose ไปได้เลย!
        childRef.value.showAlert();
    }
};
</script>
```

---

## 💡 ข้อควรระวัง

* เราควรเลือกใช้ Expose เฉพาะกับฟังก์ชันจัดการพฤติกรรมบางอย่าง (Behavior) ที่ "จำเป็น" ต้องถูกกระตุ้นเป็นครั้งๆ จากภายนอกเท่านั้น (Imperative Paradigm)
* การใช้คู่หู **Props/Emit** ยังคงเป็นรูปแบบทางเดินของข้อมูลที่ได้มาตรฐาน (Declarative Paradigm) ซึ่งถือว่าเป็นเครื่องมือลำดับแรกสุดที่คุณควรนึกถึงในการออกแบบสื่อสารข้ามคอมโพเนนต์เสมอครับ
