# 🔄 defineModel (Two-way Data Binding)

โดยปกติใน Vue.js (และ Nuxt 3) การส่งข้อมูลระหว่างคอมโพเนนต์จะยึดหลัก **One-way Data Flow** คือ ข้อมูลไหลจากแม่ไปสู่ลูก (ผ่าน "Props") และถ้าลูกต้องการอัปเดตข้อมูลนั้น จะต้องตะโกนบอกแม่ (ผ่าน "Emit") เพื่อให้แม่เป็นคนแก้ไขเอง

แต่ในหลายๆ กรณีที่ต้องสร้างช่องกรอกข้อมูล เช่น Form Inputs (Input Text, Checkbox, Dropdown) การต้องมาเขียนประกาศรับ Props และคอยยิง Emit คืนกลับทุกครั้งทำให้เกิดโค้ดที่ซ้ำซ้อนและน่าเบื่อหน่าย 

Vue เลยมีคำสั่ง `v-model` ออกมาให้ และในเวอร์ชัน Vue 3.4+ ก็ได้เปิดตัวฮีโร่ตัวช่วยล่าสุดที่ชื่อว่า **`defineModel()`** ซึ่งเป็นคำสั่งที่ช่วยให้เราทำตัวแปรแบบ **ส่งเข้าและส่งกลับในตัวเดียวกัน (Two-way Binding)** ได้ทันทีจากฝั่งลูก!

---

## 1. การทำงานของ `defineModel`

`defineModel()` จะทำหน้าที่สร้างตัวแปรชนิด Ref ขึ้นมาหนึ่งตัวในฝั่ง Child Component ซึ่งเมื่อใดก็ตามที่ตัวแปรฝั่งนี้ถูกปรับเปลี่ยนค่า (เช่น ถูกผู้ใช้พิมพ์ข้อความกรอกลงไป) ตัวทำงานเบื้องหลังของ Vue จะแอบไปจัดการมัดรวมและส่ง Emit ขึ้นไปหา Component แม่เพื่ออัปเดตค่าให้โดยอัตโนมัติแบบไร้รอยต่อ

> สรุปง่ายๆ คือ **"แก้ข้อมูลที่ลูก ก็สะเทือนและโอนถ่ายข้อมูลถึงแม่ทันทีโดยที่เราไม่ต้องเขียนฟังก์ชัน Emit เอง"**

---

## 2. ตัวอย่างการใช้งานจริง

สถานการณ์จำลอง: เราสร้างคอมโพเนนต์ลูกขึ้นมา ให้ทำหน้าที่เป็นแบบฟอร์มให้กรอกข้อมูล แล้วรอให้ค่าอัปเดตตีกลับไปที่แม่

### `app/pages/defineModal/childDefineModel.vue`
ในฝั่งของลูก (Child) ให้ลบเรื่อง `defineProps` และ `defineEmits` ออกไปเลยแล้วใช้แค่ `defineModel` ระบุชื่อที่ต้องการ
```vue
<script setup lang="ts">
// 1. ประกาศตัวแปรรับค่าและคืนค่าพร้อมกันด้วย defineModel
// ในวงเล็บ 'text' คือการระบุชื่อ Model ไว้ให้แม่รู้
const textModel = defineModel<string>('text')
</script>

<template>
  <div>
    <!-- 2. ผูกโมเดลเข้ากับช่องกรอกข้อมูลของลูก เมื่อกรอกเสร็จ v-model ของ HTML จะจัดการจับการพิมพ์ แล้วส่งค่าอัปเดตกลับไปที่ textModel ให้อัตโนมัติและแอบส่งให้เรารวย -->
    <input type="text" v-model="textModel" placeholder="ลองพิมพ์อะไรบางอย่างที่นี่สิ (ในฝั่งลูก) ..." />
  </div>
</template>
```

ทีนี้ ในคอมโพเนนต์แม่ เราก็นำคอมโพเนนต์ลูกที่เราเพิ่งสร้างไปเรียกใช้งาน และระบุตัวเชื่อมแบบ **`v-model:ชื่อโมเดล="..."`**

### `app/pages/defineModal/index.vue`
```vue
<script setup lang="ts">
import { ref } from 'vue';
import ChildDefineModel from './childDefineModel.vue';

// 1. สร้างตัวแปรเก็บสถานะตั้งต้นในฝั่งแม่
const sharedText = ref('');
</script>

<template>
    <!-- 2. เราส่ง sharedText เข้าไปผ่านการเชื่อมต่อแบบ v-model:text -->
    <!-- เมื่อลูกแก้ไขเสร็จ sharedText ฝั่งแม่ก็จะเปลี่ยนตามอัตโนมัติแบบ Realtime! -->
    <ChildDefineModel v-model:text="sharedText" />
    
    <p>ข้อมูลปัจจุบันที่แม่เห็น: {{ sharedText }}</p>
</template>
```

## 💡 ประโยชน์โดยสรุป

การใช้ `defineModel` ช่วยลดภาระความยุ่งยากของการเขียนแบบ Props + Emit แบบดั้งเดิมได้อย่างมหาศาล ทำให้การสร้าง Custom Component จำพวก Form Element ไม่พะรุงพะรัง สามารถอ่านโค้ดได้รวดเร็วและสะอาดตาขึ้นอย่างเป็นมาตรฐานยุคใหม่ครับ
